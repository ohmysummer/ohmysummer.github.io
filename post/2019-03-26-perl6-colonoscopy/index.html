Perl 6 Colonoscopy

Perl 6 中的冒号

  Posted on March 26, 2019  |  4 minutes (818 words)

Perl 6 Colonoscopy

我保证，这不会伤害你。太多…

在本文中，我们将研究 Perl 6 中的冒号，把它们放置在哪里以及它们的含义。

Sigils

但我们先绕个弯。在处理变量时，Perl 6 领域的新手不能错过 sigils 的重要性。他们是：

  • $ (Anything)
  • @ (Array)
  • % (Hash)
  • & (Callable)

sigil 是变量名的一部分，并且不会根据上下文发生变化（如 Perl 5 中所示）：

What Perl 5 Perl 6 Perl 6 Alternate Syntax
Array @a @a
Array element $a[0] @a[0]
Array slice @a[0,1] @a[0,1]
Hash %h %h
Hash element $h{'a'} %h{'a'} %h<a>
Hash slice @h{'a', 'b'} %h{'a', 'b'} %h<a b>

Perl 6 还有 Twigils，它是 igil 和变量名之间的二级字符，其稍微改变了变量的含义。我们不会在本文中介绍（有一个例外，冒号）。

有关变量，Sigil 和 Twigils 的更多信息，请参阅 [docs.perl6.org/language/variables](https://docs.perl6.org/language/variables）。

命名空间

双冒号(::)用于处理另一个名称空间中的名称（变量，过程或类）。

假设我们有一个模块 Foo，带有一个过程 bar 和一个变量 $bar。我们可以像这样访问它们（假如模块使它们在外部代码中可用）：

use Foo;

Foo::bar;       # A procedure call

say $Foo::bar;  # Accessing a variable

我们也可以直接使用命名空间（但不建议这样做）：

File: namespace

my $a = 10;
my $FOO::BAR::a = 20;

say $a + $FOO::BAR::a;  # -> 30

两个冒号也在自己的工作上，并让我们访问有关当前作用域的元信息（据我所知）：

Two colons on their own work as well, and gives us access to meta information about the current scope (as far as I understand it):

> say ::
PseudoStash.new((!UNIT_MARKER => (!UNIT_MARKER), $=pod => [],
  $?PACKAGE => (GLOBAL), $_ => (Any), ::?PACKAGE => (GLOBAL),
  EXPORT => (EXPORT), GLOBALish => (GLOBAL)))

有关 PseudoStash 类型的信息，请参阅 [docs.perl6.org/type/PseudoStash](https://docs.perl6.org/type/PseudoStash），但要准备好（或保持）混淆。

Versioned Modules

模块可以具有可选的版本和/或作者值（以及其他的一些东西）。它们在模块名称后面加上一个冒号，如下所示：

module Foo:ver<1.00>:auth { ... }

Inline::Perl5

该模块可以使用 Perl 5 模块。它是间接使用的，其中 «Foo::Bar» 是一个 Perl 5 模块：

use Foo::Bar:from<Perl5>;

这也适用于其他语言。有关可用语言的列表，请参阅 modules.perl6.org/search/?q=inline。支持可能不完整，请阅读文档。

标签

标签只是一个标签。它本身没有任何作用。

我们不能用它在代码中跳转，因为 Perl 6 没有 goto 命令。但我们可以使用它来跳出循环（或其中的循环），使用 last，next 或 redo 命令。

不使用标签考虑 last 命令：

File: last

for 1 .. 10 -> $a
{
  for 1 .. 00 -> $b
  {
    last if $b == 5;
    say "$a $b";
  }
}

如果满足条件，则 last 将退出内循环。如果我们使用标签，我们可以让它在另一个层面上行动：

File: last2

outer: for 1 .. 10 -> $a
{
  inner: for 1 .. 10 -> $b
  {
    last outer if $b == 5;
    say "$a $b";
  }
}

这次它在满足条件时退出外部循环（和程序，因为没有更多的代码可以执行）。

大写字母通常用于标签，但如果您碰巧选择 Perl 6 内部已使用的名称，这将导致错误。标签本身不是问题，因为尾部冒号告诉编译器它是一个标签。用作裸字的用法会导致问题，并且无法修复它。

一些例子：

  • OUTER，作为标签名称似乎完全合乎逻辑，将给出运行时错误（«无法解析下一个调用者（OUTER:U）; …»），因为 OUTER 是内置包名称而 Perl 6 不是聪明到足以发现它在这种情况下被用作标签。
  • LAST 使用起来并不是很诱人，但是会给出编译错误（«Missing block»），因为 LAST 是一个 *Phaser*，期待它之后有一个块。

有关要避免的包名称的更多信息，请参阅 docs.perl6.org/language/packages#Pseudo-packages 。

有关Phasers的更多信息，请参见 docs.perl6.org/language/phasers。

如果在使用标签时收到错误消息，即使您已避免使用保留的 packagae 名称和 Phasers，请尝试通过以下方式更改名称：附加一个数字。或者使用小写字母。

数字

我们可以在任何基数（从2到16）指定数字，如下所示;

my $b = :2<11110000>;  # -> 240   ## A binary (base 2) number
my $a = :16<FF>;       # -> 255   ## A hexadecimal (base 16) number

请注意，如果我们想让它看起来像过程调用，我们可以使用圆括号和引号：

my $b = :2("11110000");  # -> 240   ## A binary (base 2) number

Pair

pair 是单个键和值。它们是 Hashes 和命名参数的构建块（我们将返回）。

我们可以用几种方法构造一个 Pair，使用 «fat arrow»(=>)：

Pair('key' => 'value')
('key' => 'value')
'key' => 'value'
key => 'value'    # As long as the key doesn't contain spaces.

我们不必使用«胖箭头»：

Pair.new('key', 'value')

只要键不包含空格，我们就可以使用特殊的冒号语法：

:key<value>  # The same as «key => 'value'»

我们也可以使用圆括号：

:foo(127) # The same as «foo => 127»

如果键使用字母而值仅使用数字，我们可以这样做：

:127foo  # The same as  «foo => 127»

如果值是布尔值，我们可以进一步缩短它：

:key    # The same as «key => True»
:!key   # The same as «key => False»

变量

我们可以在三个可能的位置向变量添加冒号:

  • 在 sigil 之前(:$variable)
  • 在 sigil 和变量名之间($:variable)
  • 在变量名之后 ($variable:)

他们都是合法的。

:$variable

这是一个在过程调用中使用的命名变量（或者一个 Pair 构造函数，如本节末尾所述）：

sub foo (:$one, :$two)
{
  return $one + 2 * $two;
}

say foo(one => 12, two =>  3);  # -> 18
say foo(two =>  3, one => 12);  # -> 18

顺序无关紧要。

如果我们给变量赋予*相同名称*作为参数，我们可以使用一个简短形式（显示在最后一行）：

sub foo (:$one, :$two)
{
  return $one + 2 * $two;
}

my $one = 12;
my $two = 3;

say foo(one => $one, two =>  $two);  # -> 18
say foo(:$one, :$two);               # -> 18

当我们有几个参数时，命名变量很有用，因为可以将顺序与正常*位置参数*混淆。当我们有可选参数时，它也很方便，因为我们可以只指定我们想要使用的参数。

如果命名参数的值为“True”，我们可以删除显式值：

sub debug (:$debug)
{
  say "Debug" if $debug;
  return 32;
}

my $x = foo(:debug);

我们可以否定它（冒号和名字之间带有感叹号），传递 ‘False`：

my $x = foo(:!debug);

默认情况下，命名变量是可选的，但我们可以通过在过程头中添加一个尾随的 !（感叹号）来强制它们。

我们可以将任何变量转换为 Pair，变量名作为键：

my $age = 14;
my $p = :$age; 
say $p; # -> age => 10

$:variable

这是一个命名的占位符变量，在我们跳过声明过程头中的参数的过程中使用：

sub foo
{
  return $:one + 2 * $:two;
}

say foo(one => 12, two => 3);  # -> 18

请注意，名称必须匹配 - 并且只有在没有签名（参数列表）的情况下指定过程时它才会起作用！

这是冒号是 Twigil 的那个。

(我们也可以使用*占位符变量*作为位置参数，使用 Twigil。参见 docs.perl6.org/language/variables#The_^_twigil 了解更多信息。)

$variable:

如果我们有一个对象 $foo 并想在其上调用方法 bar，通常的方法是：

$foo.bar;

但我们也可以使用更多类似语法的过程，如下所示：

bar($foo:);

带参数的方法调用

如果我们有一个对象 $foo 并且想要用一些参数调用它上面的方法 bar，通常的方法是：

class foo
{
  ...;
    
  method bar ($abc, $def) { ... }
}
    
my $foo1 = foo.new;

$foo1.bar(1, 2);

不可能跳过圆括号（我们可以通过过程调用），但我们也可以在这里使用特殊的*冒号语法*：

$foo1.bar: 1, 2;

像语法这样的过程也适用，但要带上圆括号：

bar($foo1: 1, 2);

:= (Binding)

普通变量是一个*命名容器*（也称为桶），它可以包含一个或多个值，具体取决于类型。

绑定会跳过容器，并直接使用该值。在值上使用时，这会给出一个常量：

my $a := 10;

say $a;  # -> 10;

$a++;    # Fails.
Cannot resolve caller postfix:<++>(Int:D); the following candidates
match the type but require mutable argument ...

如果我们绑定到另一个变量，我们得到一个别名：

my $a = 10;

my $b := $a; say $b;  # -> 10;
$a++; say $b;         # -> 11;
$b++; say $a;         # -> 12;

$b 自增是有效的，因为它已被绑定到我们从 $a 获得的容器上。

=:= (容器恒等)

如果两个参数都绑定到同一个容器，则该操作符返回 True：

my $a = 10;

my $b  = 10; say $a =:= $b;  # -> False
my $c := $a; say $a =:= $c;  # -> True

即使我们没有任何容器它也工作（因为它们绑定到*同一个值*上）：

my $a := 10; my $b := 10;

say $a =:= $b;  # -> True

::?CLASS

这个*编译时*变量告诉我们我们所在的类的名称：

File: class-doublecolon

class FOO
{
  method bar { return ::?CLASS; }
}

my FOO $foo = FOO.new;

say $foo.bar;  # -> (FOO)

如果我们不在类里面，它就会失败：

say ::?CLASS;
===SORRY!=== Error while compiling:
No such symbol '::?CLASS'

::($variable)

使用此语法在运行时将字符串转换为变量名称：

File: variable-doublecolon

my ($a1, $a2, $a3, $a4, $a5, $a6, $a7, $a8, $a9, $a10) = <12 9 8 27 75 5 18 21 36 99>;

say "$_: ", ::{ '$a' ~ $_ } for 1 .. 10;

单引号是必不可少的，因为我们绝对不希望对字符串进行插值（因为不存在）。

如果要检查变量是否存在（已定义），请使用 defined，因为 «normal» 查找不会失败：

say ::{'$a11'};          # -> Nil
say ::{'$a11'}.defined;  # -> False
say ::{'$a10'}.defined;  # -> True

我们也可以用它来查找其他作用域内的变量。有关此主题的更多信息，请参阅docs.perl6.org/language/packages#Looking_up_names。
