Raku Colon Pair

Everything to know about colonpairs in Raku

  Posted on June 19, 2018  |  7 minutes (1448 words)

原文

如果我选择 Raku 编程语言中最普遍的构造，那肯定是 colonpair。哈希构造函数，命名参数和参数，副词和正则表达式修饰符 - 都涉及到 colonpair。毫不奇怪，在这样广阔的空间里，构建 colonpair 时会有很多捷径。

今天，我们将了解所有这些！这样做会让我们看到最简单的以及一些更高级的语言结构，所以如果本文的某些部分让你头脑发热，不要担心 - 你不必一次就知道所有的东西！

第一部分: 创建

Colonwhaaaa?

Colonpair 的名字（通常）来自于 Pair 对象构造函数，并且（通常）里面有一个冒号。以下是一些 colonpairs 的例子：

:foo,
:$bar,
:meow<moo>,
heh => hah

最后一个没有冒号，但由于它和其他 colonpairs 基本上是一样的，我个人认为它也是一个 colonpair。

我们可以通过查看它们的 0 来看到这些 colonpairs 构成 Pair 对象：

say :foo.^name; # OUTPUT: «Pair␤»

但是，在参数列表中使用时，这些 colonpairs 被特殊处理了以表示命名参数。我们将在后面的文章中介绍这一部分。

简写

这里有一个完整的可用的 colonpair 列表，你可以在深入之前浏览一下。我知道，它看起来像一个巨大的列表，但这就是我们阅读这篇文章的原因 - 要了解构成这些排列的一般模式。

# 标准的, take-any-type, 非便捷形式
:nd(2).say;             # OUTPUT: «nd => 2␤»
:foo('foo', 'bar').say; # OUTPUT: «foo => (foo bar)␤»
:foo( %(:42a, :foo<a b c>) ).say;
# OUTPUT: «foo => {a => 42, foo => (a b c)}␤»

# 也可以使用胖箭头记法:
# (parentheses around them are here just for the .say call)
(nd => 2).say;                      # OUTPUT: «nd => 2␤»
(foo => ('foo', 'bar') ).say;       # OUTPUT: «foo => (foo bar)␤»
(foo => %(:42a, :foo<a b c>) ).say; # OUTPUT: «foo => {a => 42, foo => (a b c)}␤»

# 布尔
:foo .say;   # OUTPUT: «foo => True␤»
:!foo.say;   # OUTPUT: «foo => False␤»

# 无符号整数
:2nd   .say; # OUTPUT: «nd => 2␤»
:1000th.say; # OUTPUT: «th => 1000␤»

# Strings and Allomorphs (看起来像数字的字符串是 Str + numeric 类型)
:foo<bar>      .say; # OUTPUT: «foo => bar␤»
:bar<42.5>     .say; # OUTPUT: «bar => 42.5␤»
:bar<42.5>.perl.say; # OUTPUT: «:bar(RatStr.new(42.5, "42.5"))␤»

# Positionals
:foo['foo', 42.5] .say; # A mutable Array:   OUTPUT: «foo => [foo 42.5]␤»
:foo<foo bar 42.5>.say; # An immutable List: OUTPUT: «foo => (foo bar 42.5)␤»
# angled brackets give you allomorphs!

# Callables
:foo{ say "Hello, World!" }.say;
# OUTPUT: «foo => -> ;; $_? is raw { #`(Block|82978224) ... }␤»

# Hashes; keep 'em simple so it doesn't get parsed as a Callable
:foo{ :42a, :foo<a b c> }.say; # OUTPUT: «foo => {a => 42, foo => (a b c)}␤»

# Name and value from variable
:$foo;  # same as :foo($foo)
:$*foo; # same as :foo($*foo)
:$?foo; # same as :foo($?foo)
:$.foo; # same as :foo($.foo)
:$!foo; # same as :foo($!foo)
:@foo;  # same as :foo(@foo)
:@*foo; # same as :foo(@*foo)
:@?foo; # same as :foo(@?foo)
:@.foo; # same as :foo(@.foo)
:@!foo; # same as :foo(@!foo)
:%foo;  # same as :foo(%foo)
:%*foo; # same as :foo(%*foo)
:%?foo; # same as :foo(%?foo)
:%.foo; # same as :foo(%.foo)
:%!foo; # same as :foo(%!foo)
:&foo;  # same as :foo(&foo)
:&*foo; # same as :foo(&*foo)
:&?foo; # same as :foo(&?foo)
:&.foo; # same as :foo(&.foo)
:&!foo; # same as :foo(&!foo)

让我们分解这些并仔细观察！

标准的，任意类型，非快捷方式

“标准”形式的冒号对由冒号(:)组成，该冒号（:)是用作所创建的 Pair 对象的 .key 的有效术语，然后其中的一组括号是具有 .value 的表达式的 Pair：

:nd(2).say;                       # OUTPUT: «nd => 2␤»
:foo('foo', 'bar').say;           # OUTPUT: «foo => (foo bar)␤»
:foo( %(:42a, :foo<a b c>) ).say; # OUTPUT: «foo => {a => 42, foo => (a b c)}␤»

只要 key 是一个有效的标识符，所有其他形式的 colonpairs 都可以用这种方式编写。对于无效的标识符，您可以简单地使用 .new 方法 – Pair.new('key'，'value') — 或”胖箭头”语法。

胖箭头语法

如果您曾经使用过 Perl 5，就不需要向你再介绍这种语法了：写一个 key - 如果它是一个有效的标识符，它将被自动引用，所以在这些情况下，您可以省略引号，然后写下 =>，然后写下值。如果 key 不是有效的标识符，那么胖箭头是唯一的 operator-involved 的语法，则可以让您用这些键构造 Pairs：

# (outer parentheses are here just for the .say call)
(nd => 2).say;                      # OUTPUT: «nd => 2␤»
(foo => ('foo', 'bar') ).say;       # OUTPUT: «foo => (foo bar)␤»
(foo => %(:42a, :foo<a b c>) ).say; # OUTPUT: «foo => {a => 42, foo => (a b c)}␤»
("the key" => "the value").say;     # OUTPUT: «the key => the value␤»

关于这个表单在参数列表中的表现以及无符号变量和常量有一些额外的规则，我们将在后面的文章中看到。

布尔值快捷方式

现在我们开始进入捷径！最常用的命名参数是什么？可能要数指定布尔标志。

总是必须将这些写为 :foo(True) 应该是相当恼人的，所以有一条捷径：完全忽略掉值，如果你想要 :foo(False)，则省略该值，并将取反运算符放在冒号后面：

# Booleans
:foo .say; # OUTPUT: «foo => True␤»
:!foo.say; # OUTPUT: «foo => False␤»

# Equivalent calls:
some-sub :foo :!bar :ber;
some-sub foo => True, bar => False, ber => True;

快捷方式的格式要短得多。这也是你可能在副词和正则表达式语法中看到的形式，例如 m// quoter 上的 :g 副词和正则表达式中的 :s / :!s 有意义的空白修饰符：

say "a b c def g h i" ~~ m:g/:s \S \S [:!s \S \s+ \S]/;
# OUTPUT: «(｢a b c d｣ ｢f g h i｣)␤»

这也是我个人提供的另一个技巧：由于 Bool 类型是一个 Int，所以可以使用布尔快捷键来指定 Int 值 1 和 0：

# set `batch` to `1`:
^4 .race(:batch).map: { sleep 1 };
say now - ENTER now; # OUTPUT: «1.144883␤»

但是，为了清晰起见，您可能希望使用无符号整数的 colonpair 快捷方式，它不是很长。

无符号整数便捷方式

当你用正则表达式匹配东西时，Raku 编程语言可以让你获得第 n 个匹配：

say "first second third" ~~ m:3rd/\S+/;
# OUTPUT: «｢third｣␤»

正如你现在猜测的那样：在 m// quoter 的 m 之后的第三个副词，用无符号整数捷径写成一个 colonpair。这种形式由冒号和 key 的名称组成，它们之间放置了不带引号的无符号整数值。没有符号，没有小数点，甚至不允许在数字之间用下划线分隔符。

这种快捷方式的主要用途是具有序号后缀的事物，例如：1st，：2nd，：9th等。它提供了很好的可读性，但个人而言，我对使用此语法的所有无符号整数值没有任何保留，无论关键的名字是。当你第一次遇到这样的语法时，它感觉稍微偏色，但它很快就会增长：

some-sub :1st :2nd :3rd :42foo :100bar;
^4 .race(:1batch).map: { sleep 1 };

Hash/Array/Callable 便捷方式

使用标准的 colonpair 格式，你可能会注意到一些格式括号太多了：

:foo(<42>)                 # value is an IntStr allomorph
:foo(<a b c>)              # value is a List
:foo([<a b c>])            # value is an Array
:foo({:42foo, :100bar})    # value is a Hash
:foo({.contains: 'meows'}) # the value is a Callable

在这些形式中，可以简单地省略外部括号并让内部括号和花括号完成他们的工作：

:foo<42>                   # value is an IntStr allomorph
:foo<a b c>                # value is a List
:foo[<a b c>]              # value is an Array
:foo{:42foo, :100bar}      # value is a Hash
:foo{.contains: 'meows'}   # the value is a Callable

它看起来更简洁，写起来更简单。 Hash 和 Callable 都使用与语言中其他地方的 {...} 构造相同的简单规则：如果内容为空，或者包含以 Pair 文字或 %-sigiled 开头的单个列表变量，并且不使用 $_ 变量或占位符参数，则创建哈希;否则会创建一个块（可调用）。

尖括号形式（:foo<...>）遵循与语言中其他地方使用的尖括号引号相同的规则：

:foo< 42  >.value.^name.say; # OUTPUT: «IntStr␤»
:foo<meows>.value.^name.say; # OUTPUT: «Str␤»
:foo<a b c>.value.^name.say; # OUTPUT: «List␤»

请记住，这两种形式并不相同：

:42foo
:foo<42>

第一个创建一个 Int 对象，而第二个创建一个 IntStr 对象，这是一个异形。这种差异对于关注对象相等性的东西很重要，例如集合运算符。

Sigiled 便捷写法

我发现在其他语言中编写代码时很痛苦的一件事就是这样的结构：

my $the-thing-with-a-thing = …
…
some-sub the-thing-with-a-thing => $the-thing-with-a-thing;

将您的变量命名为您希望将该变量作为值传递给某个命名参数的方式相当常见。 Raku 编程语言为这种情况提供了恰到好处的 colonpair 快捷方式。只需在变量名前加一个冒号来构造一个冒号对，其中键名与变量的名字相同（不包括sigil），而值则是该变量的值。唯一的问题是变量必须有一个 sigil，所以你不能在 sigilless或常量的情况下使用这个快捷方式。

my $the-thing-with-a-thing = …
…
some-sub :$the-thing-with-a-thing;

你会注意到，上面的语法看起来像是如何声明一个带有这样一个命名参数的参数 - 一致性是一件好事。支持所有可用的 sigils 和 twigils，这使得此快捷方式的完整变体列表如下所示：

# Name and value from variable
:$foo;  # same as :foo($foo)
:$*foo; # same as :foo($*foo)
:$?foo; # same as :foo($?foo)
:$.foo; # same as :foo($.foo)
:$!foo; # same as :foo($!foo)
:@foo;  # same as :foo(@foo)
:@*foo; # same as :foo(@*foo)
:@?foo; # same as :foo(@?foo)
:@.foo; # same as :foo(@.foo)
:@!foo; # same as :foo(@!foo)
:%foo;  # same as :foo(%foo)
:%*foo; # same as :foo(%*foo)
:%?foo; # same as :foo(%?foo)
:%.foo; # same as :foo(%.foo)
:%!foo; # same as :foo(%!foo)
:&foo;  # same as :foo(&foo)
:&*foo; # same as :foo(&*foo)
:&?foo; # same as :foo(&?foo)
:&.foo; # same as :foo(&.foo)
:&!foo; # same as :foo(&!foo)

这包括了当前可用的 colonpair 快捷方式列表。正如你所看到的，大量的快捷方式被简化为几个简单的模式。但是，这可能不是一直都存在的快捷方式。

未来

虽然目前尚不可用，但以下两种快捷方式可能成为未来语言版本的一部分。

第一个是间接查找快捷方式。如果你有一个命名变量, 并且该变量的名字在另一个变量中，则可以使用间接查找构造访问第一个变量的值：

my $foo = "bar";
my %bar = :42foo, :70bar;
say %::($foo); # OUTPUT: «{bar => 70, foo => 42}␤»

如果你斜视，那么间接查找就像是一个 sigely 变量，并且存在用于变量变量的结构对快捷键，所以这对于语言是一致的并且支持间接查找结果对快捷方式是有意义的，看起来像这样，其中值 $foo 包含该 colonpair 的键的名称。

:%::($foo)

这种形式目前在 R＃1532 中被列为简单的未实现的功能，所以它有可能在某一天重见天日。

第二个可能的未来结构是 :.foo 格式，它在 RFC R＃1462 中提出。这个形式在 $_ topical 变量上调用 .foo 方法，并使用返回值作为创建的 Pair 的值，方法的名称是键的名称。

当您将一个对象的属性值传递给另一个具有相似命名属性的对象时，这种形式会频繁出现，所以如下所示：

Some::Other.new: :foo(.foo) :bar(.bar) :ber(.ber) with $obj

将以这种方式写成快捷方式：

Some::Other.new: :.foo :.bar :.ber with $obj

在写这篇文章的时候，这个 RFC 已经被自我拒绝了，但是你永远不知道是否会有更多的调用来引入这个语法。

第二部分： 使用

现在我们熟悉如何编写所有形式的 colonpair，我们来看看它们的一些可用用途，特别是那些有特殊规则的用途。

参数

要指定一个参数应该是一个命名而不是位置参数，只需使用 sigilled 变量冒号对（colonpair）快捷方式：

sub meow($foo, :$bar) {
    say "$foo is positional and $bar is named";
}
meow 42, :100bar; # 42 is positional and 100 is named
meow :100bar, 42; # 42 is positional and 100 is named

由于参数需要某种变量来绑定他们的东西，几乎所有其他形式的 colonpair 都不可用于参数。这意味着，例如，你不能声明 sigilless 的命名参数，而必须明确地使用 is raw trait 来获得原始性：

sub meow (\foo, :$bar is raw) {
    (foo, $bar) = ($bar, foo)
}
my $foo = 42;
my $bar = 100;
meow $foo, :$bar;
say [$foo, $bar]; # OUTPUT: «[100 42]»

在参数中可用的另一个 colonpair 形式是用于将多个命名参数别名为相同名称和参数解构的标准形式：

sub meow (:st(:nd(:rd(:$nth))), Positional :list(($, $second, |))) {
    say [$nth, $second];
}
meow :3rd, :list<a b c>; # OUTPUT: «[3 b]»

专业提示：如果您使用的是 Rakudo 编译器，您可能希望轻松使用别名。使用超过 1 级深度的别名会导致编译器切换到慢速路径绑定器，正如名称所示，速度慢大约 10 倍。

您可以使用的技巧是使用多个参数，每个别名的最大深度为1，然后将它们合并到主体中：

sub meow (:st(:$nd is copy), :rd(:$nth)) {
    $nd //= $nth;
}

参数列表

在参数列表中使用 colonpair 应该有一个单独的部分，因为这个规则足够微妙，足以在语言陷阱部分获得一席之地。该规则涉及程序员可能希望将参数列表中的Pair 对象作为命名参数或位置参数传递的问题。

在大多数情况下，这些 colonpairs 将作为命名参数传递：

sub args {
    say "Positional args are: @_.perl()";
    say "Named      args are: %_.perl()";
}

args :foo, :50bar, e => 42;
# OUTPUT:
# Positional args are: []
# Named      args are: {:bar(50), :e(42), :foo}

要将 Pair 对象作为位置参数传递，可以执行以下任何操作：

① 用圆括号包裹整个colonpair 对 ② 在 colonpair 对上调用一些方法，比如 .self 或 .Pair ; 奇怪的东西，例如在 => 运算符上使用 R meta op 也适用 ③ 用 foo => bar 语法引起键 ④ 在 foo => bar 语法中，使用非有效标识符的键 ⑤ 把你的 Pairs 放在一个列表中并用 | “操作符”把它滑入该列表中

以下是代码形式的选项列表：

my @pairs := :42foo, :70meow;
args :foo.Pair, (:50bar), "baz" => "ber", e R=> 42, 42 => 42, |@pairs;

# OUTPUT:
# Positional args are: [:foo, :bar(50), :baz("ber"),
#   42 => 2.718281828459045e0, 42 => 42, :foo(42), :meow(70)]
# Named      args are: {}

如果您是来自其他语言（如 Perl 5），则使用胖箭头（=>）进行键/值分隔，那么编号(3)尤其值得关注。只有在键未加引号且仅当它是有效标识符时，此构造才会作为命名参数传递。

如果发生这种情况，您必须使用这些构造之一，但希望将它们作为命名参数而不是位置参数，只需将它们包裹在圆括号中并使用 | 前缀”滑入”它们。对于我们在前面的示例中已经滑过的 Pairs 的列表，你需要首先将它强制转换为 Capture 对象，因为 Pair s 填充到 Capture 中 - 与列表不同 - 作为命名参数，当 Capture 被滑入参数列表时：

my @pairs := :42foo, :70meow;
args |(:foo.Pair), |(:50bar),   |("baz" => "ber"),
     |(e R=> 42),  |(42 => 42), |@pairs.Capture;

# OUTPUT:
# Positional args are: []
# Named      args are: {"42" => 42, :bar(50),
#                      :baz("ber"), :foo(42), :meow(70)}

可以使用相同的滑动技巧来有条件地提供命名参数：

sub foo (:$bar = 'the default') { say $bar }

my $bar;
foo |(:bar($_) with $bar);  # OUTPUT: «the default␤»
$bar = 42;
foo |(:bar($_) with $bar);  # OUTPUT: «42␤»

如果 $bar 没有被定义(defined)，那么 with 语句修饰符将返回 Empty，当使用 | 滑动时将最终为空，允许参数达到其默认值。由于 |(:) 看起来像是一个侧身忍者，我称之为”忍者神兵”。

自动引起, 以 => 的形式

眼尖的观众可能已经注意到上一节中的 e => 42 colonpair 对使用字母 e 作为键，但是以相反的形式 e R=> 42，e 变成 2.718281828459045e0，因为核心语言把 e 定义作为欧拉数字。

它在 e => 42 形式中 e 仍然是一个普通字符串的原因是因为 => 构造自动引起了作为有效标识符的键，因此它们总是作为字符串，即使存在同名的常量，无信号变量或子例程：

my \meows = '🐱';
sub ehh { rand }
say %(
    meows => 'moo',
    ehh   => 42,
    τ     => 'meow',
); # OUTPUT: «{ehh => 42, meows => moo, τ => meow}␤»

有很多方法可以避免这种自动引起行为，但我会向你展示一个足够好的方法：在键的周围敲上一对括号：

my \meows = '🐱';
sub ehh { rand }
say %(
    (meows) => 'moo',
    (ehh)   => 42,
    (τ)     => 'meow',
);
# OUTPUT: «{0.58437052771857 => 42, 6.283185307179586 => meow,
#           🐱 => moo}␤»

简单！

结论

这几乎是关于 colonpair 所有的知识。我们知道它们可以用来构造 Pair 对象，用作副词，并用于指定命名参数和参数。

我们了解了各种快捷方式，例如只用于布尔值 True 的键，把否定运算符或无符号整数放在冒号和键之间以指定布尔值 False 或 Int 值。我们还了解到，如果在值的周围已经存在一组花括号，方括号或尖括号，那么 colonpair 值上的括号可以省略, 并且在变量名前添加一个冒号将会创建一个将使用该变量名称作为键的 colonpair。

在本文的后半部分，我们在指定命名参数时指定了可用的 colonpair 语法，将 colonpair 作为命名参数或位置参数传递的特点，以及如何通过将其封装在括号中来避免自动引起键。

我希望你能发掘这些信息。

-Ofun
